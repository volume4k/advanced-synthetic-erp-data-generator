# Architectural Decisions

This folder records architecture decisions that should stay stable across implementation work.

Use one file per decision. Keep each record short and focused:

- **Status**: proposed, accepted, superseded, or rejected.
- **Context**: why the decision exists.
- **Decision**: what we chose.
- **Consequences**: what this makes easier or harder.

The main architecture specification remains in `generator_vision_architecture_specification.md`. Decision records explain why specific choices were made.
